2015-11-15 | Hexo

[layout]Hello,Hexo+github

#Hexo+github搭自己的blog
好的，其实这是一篇踩坑记录。biubiubiu~

(╯‵□′)╯︵┻━┻

之前一直是用Lofter没事的时候记记东西，不过觉得Loter适合记录生活，刚好老大讲到用hexo+github搭窝的办法就想自己试试，然后就觉得如果是个喜欢折腾的人，一定会喜欢它的。
这里把我踩过的坑记下来，万一以后再搭窝啥的可以少踩两个坑。

###1.Node.js
下载安装，我是在Node.js官网下载的

大概是看到有高版本就不会选择下低版本的性格，所以我点了右边那个V5.0.0的

###2.Git
这个我之前电脑上有安装过了

用homebrew安装挺简单的

官网有说Mac可能编译遇到问题什么的要装Xcode

反正我出现了这样的错误

{ [Error: Cannot find module ‘./build/Release/DTraceProviderBindings’] code: ‘MODULE_NOT_FOUND’ }
{ [Error: Cannot find module ‘./build/default/DTraceProviderBindings’] code: ‘MODULE_NOT_FOUND’ }
{ [Error: Cannot find module ‘./build/Debug/DTraceProviderBindings’] code: ‘MODULE_NOT_FOUND’ }

解决方法：
npm加上 –no-optional再装一遍

然后用浏览器打开http://0.0.0.0:4000

###5.部署到github
啊啊啊，设计狗表示没用过github啊，(╯‵□′)╯︵┻━┻

好多教程都说这一步很简单bulabula…

然后我就找教程找教程把_config.yml里面deploy的type改成github，

然后执行hexo deploy然后就出现了一堆ERROR..

然后又一个个去查为啥，然后发现hexo版本不同配置不同啊，我的是3.1.1版本

然而很多巨巨们都没写他们用的啥版本ಥ_ಥ

好咯，有ERROR就解决ERROR咯…

这里找了很多种方法，最后是按照潘伯信巨巨的简书提供的方法

deploy:
type: git
repository: https://github.com/leopardpan/leopardpan.github.io.git
branch: master

这里还有另外踩一个坑，
使用Github Page搭建博客, 需要遵循一定的规则, 需要在github建立仓库,仓库名为

反馈INFO Deploy done: git之后

通过Github用户名.github.io就可以访问了
